    주석

    주석은 주로 코드에 대한 설명을 위해 작성되며 컴파일 시 영향을 미치지 않습니다.

    replier.reply("Hello"); //replier.reply(" World!");

    이 주석은 한 줄만을 쓸 수 있는 주석입니다. 이 소스를 실행하면 'Hello'만을 출력합니다.

    replier.reply("Hello"); 
    /*
    replier.reply(" World!");
    */

    이 주석은 여러 줄을 쓸 수 있는 주석입니다. 이 소스 또한 'Hello'만을 출력합니다.

    선언

    JavaScript에는 3가지의 선언 방법이 있습니다.



    var

    : 변수를 선언하는 동시에 값을 초기화 합니다.


    let

    : 블록 범위(scope) 지역 변수를 선언하는 동시에 값을 초기화 합니다.


    const

    : 블록 범위 읽기 전용 상수를 선언합니다.



    변수 및 상수

    변수는 값을 담는 그릇과 같습니다. 그러므로 그릇 내용물을 바꿀수도, 그릇을 비울수도, 그릇을 가지고 올 수 있는것 처럼 변수의 값을 바꿀수도, 초기화 할 수도, 변수 내용을 가지고 올 수도 있습니다.



    변수명에는 문자, 밑줄(_), 달러 기호($), 숫자가 들어갈 수 있습니다. 단, 변수명에 숫자가 먼저 올 수 없습니다.

    먼저 변수를 선언하는 방법에 대해 알아보겠습니다.

    var ItIsNull;
    var str = "It is String.";
    var num = 123;
    var bool = true;
    /*혹은*/
    let ItIsNull;
    let str = "It is String.";
    let num = 123;
    let bool = true;

    이와 같이 변수 선언에는

    var

    이나

    let

    을 사용하며 변수에는 문자열, 숫자, 불린 값 등을 추가할 수 있습니다.

    ItIsNull

    와 같이 값을 추가하지 않은 변수는 호출 시

    undefined

    를 출력하게 됩니다.



    또한, 상수 선언에는

    const

    를 사용합니다.

    const ItIsNull; //호출 시 에러 발생
    const str = "It is String.";
    const num = 123;
    const bool = true;

    ItIsNull

    와 같이 값을 추가하지 않은 상수는 호출 시

    SyntaxError

    예외를 출력하게 됩니다.



    변수와 상수를 사용하는 방법을 이렇습니다:

    var var1 = 1;
    let let1 = 2;
    const const1 = 3;

    replier.reply(var1); //1
    replier.reply(let1); //2
    replier.reply(const1); //3

    var

    ,

    let

    ,

    const

    값이 저장되어 있지 않은 최상위 속성입니다.

    자료형 변환

    JavaScript는 동적 형지정(정형) 언어입니다. 즉, 변수 선언 시 데이터 형을 지정할 필요가 없습니다. 또한 데이터 형이 스크립트 실행 도중 필요에 의해 자동으로 변환이 됩니다.

    var foo = "It is String.";

    이 같이 변수

    foo

    를 선언했습니다. 현재 변수

    foo

    는 문자열 변수입니다.

    foo = 12345678;

    그리고 같은 변수에 숫자값을 할당할 수도 있습니다. 하지만 자료형을 변환하지 않는 경우도 있습니다:

    var foo = "123" + 456; //123456

    이와 같이 다른 연산자를 포함한 식의 경우, JavaScript는 숫자 값을 문자열로 변환하지 않습니다.

    연산자

    연산자는 특정한 작업을 하기 위해 사용하는 기호입니다. 자바스크립트엔 많은 연산자가 있습니다:

  • 할당 연산자
  • 비교 연산자
  • 산술 연산자
  • 비트 연산자
  • 논리 연산자
  • 문자열 연산자
  • 조건(삼항) 연산자
  • 쉼표 연산자
  • 단항 연산자
  • 관계 연산자
  • 연산자 우선순위

    • 할당 연산자

    할당 연산자는 오른쪽 피연산자의 값을 왼쪽 피연산자에 할당합니다. 기본적인 할당 연산자는 오른쪽의 피연산자 값을 왼쪽 피연산자 값에 할당하는 등호(

    =

    ) 입니다. 즉

    x = y

    는

    y

    값을

    x

    에 할당합니다. 또한, 연산의 약칭인 복합 할당 연산자도 존재합니다.

    이름 복합 할당 연산자 뜻
    할당

    x = y

    x = y
    덧셈 할당

    x += y

    x = x + y
    뺄셈 할당

    x -= y

    x = x - y
    곱셈 할당

    x *= y

    x = x * y
    나눗셈 할당

    x /= y

    x = x / y
    나머지 연산 할당

    x %= y

    x = x % y
    왼쪽 이동 연산 할당

    x <<= y

    x = x << y
    오른쪽 이동 연산 할당

    x >>= y

    x = x >> y
    부호 없는 오른쪽 이동 연산할당

    x >>>= y

    x = x >>> y
    비트 AND 할당

    x &= y

    x = x & y
    비트 XOR 할당

    x ^= y

    x = x ^ y
    비트 OR 할당

    x |= y

    x = x | y

    비교 연산자

    비교 연산자는 피연산자들을 비교하고 비교에 따라 논리 값을 반환합니다. 피연산자들은 숫자, 문자열, 논리형, 객체 를 사용할 수 있습니다. 문자열은 유니코드 값을 사용하여 표준 사전순서를 기반으로 비교합니다. 만약 두 피연산자가 다른 형태일 경우, JavaScript는 대부분 비교를 위해 피연산자를 적절한 타입으로 변환합니다. 이런 행동은 보통 숫자로 피연산자를 숫자로 비교하는 형태로 나타납니다. (

    ===

    과

    !=

    제외.) 이런 연산자는 비교를 위해 피연산자의 형태를 적절히 바꾸려고 시도하지 않습니다. 다음 표는 샘플 코드를 통해 비교 연산자에 대하여 설명합니다:

    var var1 = 3;
    var var2 = 4;
    연산자 설명 참을 반환하는 예제
    동등 (

    ==

    )    		 피연산자들이 같으면 참을 반환합니다.

    3 == var1


    "3" == var1



    3 == '3'
    부등 (

    !=

    )    		 피연산자들이 다르면 참을 반환합니다.

    var1 != 4


    var2 != "3"
    일치 (

    ===

    )    		 피연산자들이 같고 피연산자들의 같은 형태인 경우 참을 반환합니다.

    3 === var1
    불일치 (

    !==

    )    		 피연산자들이 다르거나 형태가 다른 경우 참을 반환합니다.

    var1 !== "3"


    3 !== '3'
    ~보다 큰 (

    >

    )    		 좌변의 피연산자 보다 우변의 피연산자가 크면 참을 반환합니다.

    var2 > var1


    "12" > 2
    ~보다 크거나 같음 (

    >=

    )    		 좌변의 피연산자 보다 우변의 피연산자가 크거나 같으면 참을 반환합니다.

    var2 >= var1


    var1 >= 3
    ~보다 작음 (

    <

    )    		 좌변의 피연산자 보다 우변의 피연산자가 작으면 참을 반환합니다.

    var1 < var2


    "2" < 12
    ~보다 작거나 같음 (

    <=

    )    		 좌변의 피연산자 보다 우변의 피연산자가 작거나 같으면 참을 반환합니다.

    var1 <= var2


    var2 <= 5

    주의:

    =>

    는 연산자가 아닌, 화살표 함수를 나타내는 기호입니다.

    산술 연산자

    산술 연산자는 숫자값(리터럴 또는 변수)을 피연산자로 갖고, 하나의 숫자 값을 반환합니다. 기본적인 산술 연산자는 덧셈(

    +

    ), 뺄셈 (

    -

    ), 곱셈 (

    *

    ), 나눗셈 (

    /

    ) 입니다. 이 연산자들은 대부분의 다른 프로그래밍 언어들이 부동소수점 값을 연산하는것처럼 동작합니다. (0 으로 나누는 것은

    Infinity

    을 발생시키는 것을 기억하세요) 예를 들면:

    1 / 2; //0.5
    1 / 2 == 1.0 / 2.0; //true

    또한, 표준 산술 연산자 (+, -, *, /) 에 대해 JavaScript는 다음의 표에 나와 있는 산술 연산자를 제공합니다.

    연산자 설명 예제
    나머지 연산자 (

    %

    )    		 이항 연산자입니다. 두 피연산자를 나눈후 나머지를 반환합니다. 12 % 5 는 2를 반환합니다.
    증가 연산자 (

    ++

    )    		 단항 연산자입니다. 피연산자에 1을 더합니다. 만약 연산자를 피연산자 앞(

    ++x

    )에 사용하면, 피연산자에 1을 더한 값을 반환합니다.
    만약 연산자를 피연산자 뒤(

    x++

    )에 사용하면, 피연산자에 1을 더하기 전 값을 반환합니다.

    x

    가 3이면

    ++x

    는

    x

    를 4로 만들고 4를 반환합니다. 반면

    x++

    는 3을 반환하고

    x

    를 4로 만듭니다.
    감소 연산자 (

    --

    )    		 단항 연산자입니다. 피연산자로 부터 1을 뺍니다. 반환값은 증가 연산자와 유사합니다.

    x

    가 3이면

    --x

    는

    x

    를 2로 만들고 2를 반환합니다. 반면

    x--

    는 3을 반환하고

    x

    를 2로 만듭니다.
    단항 부정 연산자 (

    -

    )    		 단항 연산자 입니다. 피연산자의 반대값(부호 바뀐값)을 반환합니다.

    x

    가 3이면

    -x

    는 -3을 반환합니다.
    숫자화 연산자 (

    +

    )    		 단항연산자 입니다. 피연산자가 숫자값이 아니라면 피연산자를 숫자로 변환하기를 시도합니다.

    +"3"

    은

    3

    을 반환합니다.

    +true

    는

    1

    을 반환합니다.

    비트 연산자

    비트 연산자는 피연산자를 10진수, 16진수, 8진수처럼 취급하지 않고 32비트의 집합으로 취급합니다. 예를 들면, 10진수의 9는 2진수의 1001로 나타낼 수 있습니다. 비트 단위 연산자는 이진법으로 연산을 수행하지만, JavaScript의 표준 숫자값을 반환합니다.

    다음의 표는 JavaScript의 비트단위 연산자의 요약입니다.

    연산자 사용법 설명
    비트단위 논리곱

    a & b

    		 두 피연산자의 각 자리 비트의 값이 둘 다 1일 경우 해당하는 자리에 1을 반환합니다.
    비트단위 논리합

    a | b

    		 두 피연산자의 각 자리 비트의 값이 둘다 0일 경우 해당하는 자리에 0을 반환합니다.
    비트단위 배타적 논리합

    a ^ b

    		 두 피연산자의 각 자리 비트의 값이 같을 경우 해당하는 자리에 0을 반환합니다.
    [두 피연산자의 각 자리 비트의 값이 다를 경우 해당하는 자리에 1을 반환합니다.]
    비트단위 부정

    ~ a

    		 피연산자의 각 자리의 비트를 뒤집습니다.
    왼쪽 시프트

    a << b

    		 오른쪽에서 0들을 이동시키면서, a의 이진수의 각 비트를 b비트 만큼 왼쪽으로 이동시킨 값을 반환합니다.
    부호 전파 오른쪽 시프트

    a >> b

    		 사라지는 비트를 버리면서, a의 이진수의 각 비트를 b비트 만큼 이동시킨 값을 반환합니다.
    부호 없는 오른쪽 시프트

    a >>> b

    		 사라지는 비트를 버리고 왼쪽에서 0을 이동시키면서, a의 이진수의 각 비트를 b비트 만큼 이동시킨 값을 반환합니다.

    예를 들면, 9의 이진표현은 1001이고, 15의 이진표현은 1111입니다. 따라서, 비트단위 연산자가 이 값들에 적용될때, 결과는 다음과 같습니다:

    조건 (삼항) 연산자

    조건 (삼항) 연산자 는 JavaScript에서 3개의 항을 사용하는 유일한 연산자 입니다. 조건 연산자는 조건에 따라 2개의 값중 하나를 가질 수 있습니다. 문법은 다음과 같습니다:

    조건 ? 참일 때 코드 : 거짓일 때 코드;

    조건이 참일 시 참일 때 코드를 실행하고 거짓일 시 거짓일 때 코드를 실행합니다.

    예를 들어,

    10 > 3 ? replier.reply("참") : replier.reply("거짓");

    10 > 3

    은

    true

    이므로 이를 실행하면 '참'을 출력하게 됩니다. 또한,

    var result = 5 > 7 ? "참" : "거짓"; // result의 값: '거짓'

    이와 같이 변수에 할당을 할 수도 있습니다.

    • if문

    if문은 괄호 안의 식이 참일 때 코드를 실행합니다. 카카오톡 봇의 명령어를 개발할 때 if문을 주로 사용합니다. if문을 사용해 명령어를 만드는 예제입니다:

    if (msg == "/cmd") { // 메시지가 '/cmd'일 때,
        replier.reply("명령어를 수행."); // '명령어를 수행.' 이라고 출력.
    }

    또는 이렇게 입력할 수 있습니다:

    if (msg == "/cmd") replier.reply("명령어를 수행."); // 메시지가 '/cmd'일 때, '명령어를 수행.' 이라고 출력.

    수많은 메시지가 수신될 때 마다 봇은 메시지가 어떤 내용인지 감지하고 이 소스를 실행시킵니다. 그러다 '/cmd' 라는 메시지를 감지하고 이 소스를 실행시키면

    msg == "/cmd"

    의 값이

    true

    가 되어 if문 안의 소스를 실행시킵니다. 결국, 봇은 '명령어를 수행.'을 출력합니다.



    반면, else는 if문의 조건이 거짓일 때 코드를 실행합니다. (else는 if보다 먼저 나올 수 없습니다.)

    if (msg == "/cmd") { // 메시지가 '/cmd'일 때,
        replier.reply("명령어를 수행."); // '명령어를 수행.' 이라고 출력.
    } else { // 메시지가 '/cmd'가 아니라면,
        Api.makeNoti("명령어가 입력되지 않았습니다.", "명령어가 입력되지 않아 noti를 띄웁니다.", 1); // 메시지 수신을 하지 않고 상단바에 노티를 띄움.
    }

    또는 이렇게 입력할 수 있습니다:

    if (msg == "/cmd") replier.reply("명령어를 수행."); // 메시지가 '/cmd'일 때, '명령어를 수행.' 이라고 출력.
    else Api.makeNoti("명령어가 입력되지 않았습니다.", "명령어가 입력되지 않아 noti를 띄웁니다.", 1); // 메시지가 '/cmd'가 아니라면, 메시지 수신을 하지 않고 상단바에 노티를 띄움.

    이 소스를 실행시킬 경우, 명령어(/cmd)가 아닌 다른 메시지가 수신되었을 때 상단바에 노티를 띄웁니다.



    else if문은 조건마다 다른 코드를 실행하고자 할 때 사용됩니다.

    if (msg == "/cmd") { // 메시지가 '/cmd'일 때,
        replier.reply("명령어를 수행."); // '명령어를 수행.' 이라고 출력.
    }
    else if (msg == "/cmd2") { // 메시지가 '/cmd2'일 때,
        replier.reply("명령어2를 수행."); // '명령어2를 수행.' 이라고 출력.
    }

    또는 이렇게 입력할 수 있습니다:

    if (msg == "/cmd") replier.reply("명령어를 수행."); // 메시지가 '/cmd'일 때, '명령어를 수행.' 이라고 출력.
    else if (msg == "/cmd2") replier.reply("명령어2를 수행."); // 메시지가 '/cmd2'일 때, '명령어2를 수행.' 이라고 출력.

    '/cmd'를 입력하면 '명령어를 수행.' 이라 출력합니다. 그리고 '/cmd2'를 입력하면 '명령어2를 수행.' 이라 출력합니다. 카카오톡 봇의 여러 명령어를 구현할 때 대부분 이 방법을 사용합니다.

    한편, 여러 조건을 통해 필터링을 하려는 경우, 중첩 if문을 사용합니다. 이는 if문 안에 if문이 존재하는 형식이며 if문의 개수는 제한되어 있지 않습니다.

    if (msg == "/cmd") { // 메시지가 '/cmd'이고,
        if (sender == "tester") { // 수신자가 'tester'일 때,
            replier.reply("tester가 명령어를 입력함."); // 'tester가 명령어를 입력함.' 이라고 출력.

    if (msg == "/cmd" && sender == "tester") { // 메시지가 '/cmd'이고 수신자가 'tester'일 때,

    switch-case문

    switch-case문은 어떤 식이나 변수 등의 값에 따라 다른 코드를 실행합니다. 예를 들면,

    switch (msg) {
        case "apple" : 
            replier.reply("사과가 입력됨.");
            break;
        case "banana" : 
            replier.reply("바나나가 입력됨.");
            break;
        case "orange" : 
            replier.reply("오렌지가 입력됨.");
            break;
        default : 
            replier.reply("다른것이 입력됨.");
            break;
    }

    switch(msg)

    는

    msg

    의 값에 따른 코드를 지정하겠다는 코드입니다. 또한,

    case "apple" ~~ break;

    는

    msg

    의 값이 'apple'일 때 '사과를 입력함.' 이라고 출력하는 코드입니다.

    default: ~~ break;

    는

    case

    로 적힌 값 이외의 값 ('apple', 'banana', 'orange'를 제외한 값)에 대한 코드입니다.
    결론은, 'apple'이 수신되었을 땐 '사과가 입력됨.'을, 'banana'가 수신되었을 땐 '바나나가 입력됨.'을, 'orange'가 수신되었을 땐 '오렌지가 입력됨.'을 출력하며 그 외의 메시지가 수신되면 '다른것이 입력됨.'을 출력합니다.



    switch-case문에서의 break문은 해당 switch문을 빠져나옵니다. 즉, break문이 없는 case문의 코드는 실행되고 나서 switch문을 빠져나오지 않고 break문이 존재하는 case문이나 default문을 만날 때 까지 모든 case문을 실행합니다. 예를 들면,

    switch (msg) {
        case "apple" : 
            replier.reply("사과가 입력됨.");
        case "banana" : 
            replier.reply("바나나가 입력됨.");
        case "orange" : 
            replier.reply("오렌지가 입력됨.");
            break;
        default : 
            replier.reply("다른것이 입력됨.");
            break;
    }

    이와 같이 break문이 존재하는 곳은

    case "orange"

    입니다. 고로, 'apple'이 수신되면

    case "apple"

    부분과

    case "banana"

    부분과

    case "orange"

    부분의 코드를 실행하게됩니다.

    var i = 0;

    와 같이 변수를 0으로 초기화하는 코드가 들어갑니다. 그러나 복잡한 구문으로 표현 될 때도 있습니다. 초기화문의 대부분의 목적은 반복문의 카운터 설정입니다.


    * 조건식은 조건이 참인지를 검사합니다. 만약 조건식이 참이 되면 반복문을 실행하지만, 거짓이 될 경우 반복문을 실행시키지 않습니다.


    * 증감식은 보통 초기화문에 들어가는 변수 등의 값을 증가시키거나, 감소시키는 코드가 들어갑니다. 대부분

    i++

    이나

    i--

    와 같이 들어갑니다. 하지만 이 외의 코드도 들어갈 수 있습니다.



    이 코드는 for문의 예제입니다:

    for (var i = 0; i < 10; i++) { // 변수 i의 값을 0으로 선언; i의 값이 10보다 작으면 반복문 실행; 반복문 안의 코드가 끝난 후 i의 값을 1 증가
        replier.reply("반복문 실행: " + i + "번째"); // 변수 i의 값 출력
    }

    결과값은 이와 같습니다:

    > 반복문 실행: 0번째
    > 반복문 실행: 1번째
    > 반복문 실행: 2번째
    > 반복문 실행: 3번째
    > 반복문 실행: 4번째
    > 반복문 실행: 5번째
    > 반복문 실행: 6번째
    > 반복문 실행: 7번째
    > 반복문 실행: 8번째
    > 반복문 실행: 9번째

    또한, 초기화문, 조건문, 증감문은 생략할 수 있습니다:

    /*초기화문 생략*/

    var i = 0;

    for (; i < 10; i++) {
        replier.reply("반복문 실행: " + i + "번째");
    }
    /*조건문 생략*/

    for (var i = 0; ; i++) {
        if (i < 10) {
            replier.reply("반복문 실행: " + i + "번째");
        } else {
            break; // break문 조건문을 강제로 탈출하는 문으로 라이노 엔진 > 조건문 > break문에서 자세히 다룹니다.
        }
    }
    /*증감문 생략*/

    for (var i = 0; i < 10; ) {
        replier.reply("반복문 실행: " + i + "번째");
        i++;
    }

    초기화문, 조건문, 증감문을 모두 생략하면 무한으로 반복하게 됩니다:

    for ( ; ; ) {
        replier.reply("무한반복 실행.");
    }
    이 코드를 실행할 시 메신저봇R 앱을 강제중지 하거나 삭제해야만 멈출 수 있습니다. 또한, 이 코드는 구동기기의 속도와 발열에 영향을 줄 수 있습니다.

    while문

    while문은 특정조건이 참일 때 반복합니다. 형식은 이와 같습니다:

    while (조건문) {
        // TODO
    }

    조건문의 불린 값이 참이되면 while문 안의 코드를 실행하며, 반대로 조건문의 불린 값이 거짓이 되면 while문 다음의 코드를 실행합니다. 예를 들면,

    var i = 0;

    while (i < 6) { // 변수 i의 값이 6 미만일 때 while문 실행.
        replier.reply(i); // 변수 i의 값 출력.
        i++; // 변수 i의 값 증가.
    }

    replier.reply("while문 종료.");

    이를 실행하면 이와 같은 결과가 나타납니다:

    > 0
    > 1
    > 2
    > 3
    > 4
    > 5
    > while문 종료.

    또한, 조건문의 불린 값이 참이되면 실행하는 while문의 특징을 이용하여 무한으로 반복하게 할 수 있습니다:

    while (true) {
        replier.reply("무한반복 실행.");
    }
    이 코드를 실행할 시 메신저봇R 앱을 강제중지 하거나 삭제해야만 멈출 수 있습니다. 또한, 이 코드는 구동기기의 속도와 발열에 영향을 줄 수 있습니다.

    중첩 for문

    for (var i = 2; i < 10; i++) { // 2단~9단을 출력. (예: 'i' × 1)

        }
        replier.reply("\n"); // 한 단이 끝나면 줄바꿈 추가.
    }

    > 2 × 2 = 4
    > 2 × 3 = 6
    > 2 × 4 = 8
    > 2 × 5 = 10
    > 2 × 6 = 12
    > 2 × 7 = 14
    > 2 × 8 = 16
    > 2 × 9 = 18

    > 3 × 1 = 3

    이와 같이 중첩 for문은 가장 외부의 for문 부터 실행합니다.

    break문

    break문은 switch-case문에서는 해당 switch문을 빠져나오는 기능을 수행하지만, 반복문에서는 해당 반복문을 빠져나오는 기능을 수행합니다. 예를 들면,

    var arr = ["Python", "Kotlin", "Java", "JavaScript", "C#"];

    for (var i = 0; i < arr.length; i++) {
        if (arr[i] == "JavaScript"){ // 만약 배열 arr의 i번째 값이 'JavaScript'라면,
            replier.reply("JavaScrirpt 단어를 검색함."); // 'JavaScrirpt 단어를 검색함.'라고 출력.
            break; // 배열 arr에서 'JavaScript' 단어를 찾았으므로 반복문 탈출.
        } else {
            replier.reply("JavaScript 단어를 찾지 못함."); // 'JavaScript' 단어를 찾지 못했으므로 'JavaScript 단어를 찾지 못함.'을 출력.
        }
    }
    replier.reply("반복문 마침."); // 반복문이 끝나면 출력.

    실행결과는 이와 같습니다:

    > JavaScript 단어를 찾지 못함.
    > JavaScript 단어를 찾지 못함.
    > JavaScript 단어를 찾지 못함.
    > JavaScrirpt 단어를 검색함.
    > 반복문 마침.

    첫번째, 두번째, 세번째의 출력값은 각각 'Python', 'Kotlin', 'Java'와 비교했을 때의 출력값입니다. 그리고 네번째 출력값은 'JavaScript'와 비교했을 때의 출력값입니다. 'JavaScript'와 비교했을 때 반복문 안의 if문이 작동하여 반복문을 빠져나온 것입니다.

    continue문

    continue문은 해당 반복문에서, 자신 다음의 코드를 무시하고 바로 증감식을 실행시킵니다. 예를 들면,

    for (var i = 0; i < 11; i++) {
        if (i % 2 == 0){ // 만약 i와 2의 나눗셈의 몫이 0이라면 (i가 짝수라면),
            continue; // 아래의 코드(i의 값을 출력하는 코드)를 무시.
        }
        replier.reply(i);
    }

    실행결과는 이와 같습니다:

    > 1
    > 3
    > 5
    > 7
    > 9

    레이블_이름 : 
    반복문

    예를 들면,

    loop :

    do {

    var i = 0;

    do {
        i += 2; // i의 값을 2 증가시킴.

    > 2
    > 4
    > 6
    > 8

    for-in문

    for-in문은 어떤 객체의 속성 개수나 배열의 길이만큼 특정코드를 반복합니다. 형식은 이와 같습니다:

    for (변수명 in 객체/배열){
        // TODO
    }

    위 소스에서, '변수명'은 객체의 프로퍼티의 이름이나 배열의 인덱스 번호를 데이터로 가집니다. 우선 객체에 대한 for-in문의 예를 들면,

    var Obj = {A : "Python", B : "Java", C : "Kotlin", D : "JavaScript", E : "C#"};

    for (name in Obj){
        replier.reply("프로퍼티 이름: " + name + ", 프로퍼티 내용: " + Obj[name]);
    }

    실행결과는 이와 같습니다:

    > 프로퍼티 이름: A, 프로퍼티 내용: Python
    > 프로퍼티 이름: B, 프로퍼티 내용: Java
    > 프로퍼티 이름: C, 프로퍼티 내용: Kotlin
    > 프로퍼티 이름: D, 프로퍼티 내용: JavaScript
    > 프로퍼티 이름: E, 프로퍼티 내용: C#

    배열에 대한 예를 들면,

    var arr = ["Python", "Java", "Kotlin", "JavaScript", "C#"];

    for (index in arr){
        replier.reply("배열 인덱스: " + index + ", 인덱스에 대한 내용: " + arr[index]);
    }

    실행결과는 이와 같습니다:

    > 배열 인덱스: 0, 인덱스에 대한 내용: Python
    > 배열 인덱스: 1, 인덱스에 대한 내용: Java
    > 배열 인덱스: 2, 인덱스에 대한 내용: Kotlin
    > 배열 인덱스: 3, 인덱스에 대한 내용: JavaScript
    > 배열 인덱스: 4, 인덱스에 대한 내용: C#

    for-of문

    for-of문은 for-in문과 같이 어떤 배열의 길이만큼 특정코드를 반복합니다. 형식은 이와 같습니다:

    for (변수명 of 배열){

    var arr = ["Python", "Java", "Kotlin", "JavaScript", "C#"];

    정규식 (RegExp객체)

    정규식 (정규 표현식이라고도 함)은 문자열에 나타는 특정 문자 조합과 대응시키기 위해 사용되는 패턴입니다. JavaScript에서 정규식은 객체입니다.(RegExp객체) 따라서, 정규식에는 메서드가 있습니다. 정규식(RegExp객체)의 생성자는 이와 같습니다:

    RegExp(pattern, flags);
    매개변수 뜻

    pattern

    		 정규식을 나타내는 텍스트입니다.

    flags

    		 정규식에 추가할 플래그입니다.

    정규식(RegExp객체)는

    new

    연산자 없이 생성자로 생성할 수 있습니다.

  • 정규식 만들기

  • 정규식 사용하기

    • 정규식 만들기

    정규식을 만드는 방법은 두 가지가 있습니다:

    var reg1 = /ab+c/;
    /*또는*/
    var reg1 = new RegExp("ab+c");

    첫번째 방법은 슬래시(/)를 이용해 만드는 것이고, 두번째 방법은

    RegExp

    객체를 이용해 만드는 것입니다.

    정규식 패턴 작성하기

    정규식 패턴은

    /abc/

    와 같이 단순 문자로 구성될 수도 있고,

    /ab*c/

    또는

    /Chapter (\d+)\.\d*/

    와 같이 단순 문자와 특수 문자의 조합으로 구성될 수도 있습니다. 먼저, 단순 문자열 패턴을 알아보겠습니다.

    var reg1 = /apple/;

    reg1

    은 어떤 문자에 포함된 'apple' 단어에 대응됩니다. 예를 들면,

    reg1

    은 'This is apple'의 'apple'에 대응되지만, 'This is a pple'에는 대응되지 않습니다.



    다음은 정규식에서 사용되는 특수기호와 그 기능에 대한 설명입니다:

    문자 뜻

    \

    		 특수문자가 아닌 문자가 뒤에 있을 때, 백슬래시(

    \

    )는 '해당 문자는 특별하므로 문자 그대로 해석되어야 한다'라는 것으로 해석합니다.
    예를 들어, 앞에

    \

    가 없는

    n

    은 보통 'n'을 나타내지만,

    \n

    은 'n'과 같은 문자가 아닌 줄바꿈을 나타내는 문자로 나타나게 됩니다.

    한편, 특수문자가 뒤에 있을 경우 백슬래시는 '해당 문자는 특별하지 않으므로 문자 그대로 해석되면 된다'라는 것으로 해석합니다.
    예를 들어,

    /\*/

    은

    *

    과 마찬가지로 '*'를 나타냅니다.

    RegExp

    객체를 이용할 경우,

    \

    을

    \\

    로 작성해야 합니다.

    ^

    		 입력의 시작 문자에 대응됩니다. 예를 들면,

    /^A/

    는 'It is Apple.'의 'A'에 대응되지 않습니다. 그러나 'Apple is delicious.'의 'A'에는 대응됩니다.

    '

    ^

    '가 문자셋 패텅의 첫글자로 쓰일 때는(

    [^abc]

    와 같을 때), 전혀 다른 의미를 가집니다(역 문자셋.)

    $

    		 입력의 끝 문자에 대응됩니다. 예를 들면,

    /h$/

    는 'teacher'의 'h'에 대응되지 않습니다. 그러나 'teach'의 'h'에는 대응됩니다.

    *

    		 0번 이상 반복되는 문자열에 대응됩니다. 예를 들면,

    /bo*/

    는 'A ghost booooed'의 'boooo'와 'A bird warbled'의 'b'에 대응되지만, 'A goat grunted'에는 아무 문자도 대응되지 않습니다.

    그러나,

    *

    앞의 글자가 한 글자일 경우 모든 문자열에 대응됩니다. 예를 들면,

    /z*/

    는 'JavaScript' 전체에 대응됩니다.

    +

    		 1번 이상 반복되는 문자열에 대응됩니다. 예를 들면,

    /a+/

    는 'candy'의 'a'와 'caaaaaaaandy'의 모든 'a'에 대응되지만, 'cndy'에는 아무 문자도 대응되지 않습니다.

    ?

    		 앞의 표현식이 0~1회 등장하는 부분과 대응됩니다. 예를 들어,

    /e?le?/

    은
    'e'가 0~1회 반복하고
    'l'이 온 후
    'e'가 0회 이상 반복되는 문자열에 대응됩니다. 그러므로 'engel'의 'el'과 'engle'의 'le', 그리고 'also'의 'l'에 대응됩니다.

    한편, *, +, ?, {}과 같이 가능한 한 많은 문자열을 대응시키는 패턴 뒤에

    ?

    를 붙이면 가능한 한 적은 문자열을 대응시킵니다.
    예를 들어,

    /\d+/

    (

    \d

    는 숫자 문자를 나타냄)는 '123가나다'의 '123'에 대응되지만,

    /\d+?/

    는 오직 '123가나다'의 '1'에 대응합니다.

    또한

    x(?=y)

    또는

    x(!?y)

    과 같이 사전 검증에도 사용됩니다.

    .

    		 개행 문자(줄바꿈 문자)를 제외한 모든 단일 문자와 대응됩니다. 예를 들면,

    /.n/

    은 'n'으로 끝나는 단일 문자에 대응됩니다. 'It is an apple.'의 'an'에 대응되지만, 'never'의 'ne'에는 대응되지 않습니다.

    (x)

    		 'x'에 대응되고, 그것을 기억합니다. 괄호는 '포획괄호'라고 불립니다. 예를 들면,

    /(foo) (bar)/

    는 'foo bar test'의 'foo bar'와 대응됩니다. 또한, 'foo'와 'bar'를 기억합니다.

    (?:x)

    		 'x'에 대응되고, 그것을 기억하지 않습니다. 괄호는 '비포획괄호'라고 불립니다. 예를 들면,

    /abc+/

    는 'abcabcabc'의 'abc'에 대응되지만,

    /(?:abc)+/

    는 'abcabcabc'에 대응됩니다.

    x(?=y)

    		 'y'가 뒤따라오는 'x'에 대응됩니다. 이것을 'lookahead' 라고 부릅니다. 예를 들면,

    /Hello(?=World)/

    는 'HelloProgramming'의 'Hello'에는 대응되지 않지만, 'HelloWorld'의 'Hello'에는 대응됩니다.

    x(?!y)

    		 'y'가 뒤따라오지 않는 'x'에 대응합니다. 이것을 'negated lookahead' 라고 부릅니다. 예를 들면,(위 정규식 예제의 문장과 같은 문장으로 예를 들겠습니다)

    /Hello(?!World)/

    는 'HelloWorld'의 'Hello'에는 대응되지 않지만, 'HelloProgramming'의 'Hello'에는 대응됩니다.

    x|y

    		 'x'또는 'y'에 대응됩니다. 예를 들면,

    /Android|KakaotalkBot/

    는 'Android Studio'의 'Android'와 'KakaotalkBot Studio'의 'KakaotalkBot'에 대응됩니다.

    {n}

    		 앞의 문자가 n번 반복되는 문자열에 대응됩니다. (n은 양의 정수여야 합니다.) 예를 들면,

    /e{3}/

    는 'Hello'의 'e'에 대응되지 않지만, 'Heeello'의 'eee'와 'Heeeeeeeeello'의 첫 'eee'(첫번째~세번째 'e')에 대응됩니다.

    {n,m}

    		 앞의 문자가 최소 n번에서 최대m번 반복되는 문자열에 대응됩니다.(n과 m은 양의 정수이고, n은 m이하의 값이여야 합니다.) m이 생략되는 경우, m은 ∞로 취급됩니다.
    예를 들면,

    /a{2, 4}/

    는 'candy'의 'a'에 대응되지 않지만, 'caaandy'의 'aaa'에는 대응됩니다.

    [xyz]

    		 문자셋(Character set) 입니다. 점(.)과 별표(*)같은 특수문자는 문자셋 안에서는 특수문자로 취급되지 않으므로 이스케이프시킬 필요가 없습니다. 또한, 하이픈(-)을 이용해 문자의 범위를 지정해 줄 수도 있습니다.
    예를 들면,

    /[0-7]/

    는

    /[0-7]/

    은

    /[01234567]/

    과 같습니다. 그리고 이는 '1st'의 1과 '3td'의 '3'에도 대응합니다.

    [^xyz]

    		 부정 문자셋(negated character set) 또는 보충 문자셋(complemented character set)입니다. 괄호 내부의 문자를 제외한 모든 문자와 대응됩니다. 예를 들어,

    /[^a-d]/

    는

    /[^abcd]/

    와 같으며, 'ant'의 'a'에 대응되지 않지만, 'n'에는 대응됩니다.

    [\b]

    		 백스페이스(U+0008)에 대응됩니다. (

    \b

    와 다릅니다.)

    \b

    		 단어 경계를 나타냅니다. 예를 들면,

    /\bm/

    은 'moon'의 'm'에 대응됩니다.

    /oo\b/

    은 'moon'의 'oo'에 대응되지 않습니다. 'oo'가 'moon'의 경계에 있지 않기 때문입니다.

    /on\d/

    은 'moon'의 'on'에 대응됩니다.

    \B

    		 단어 경계가 아닌 부분에 대응됩니다.

    \cX

    		 문자열 내부의 제어 문자에 대응됩니다. 여기서 X는 A에서 Z까지의 문자 중 하나입니다.

    \d

    		 숫자 문자에 대응되며,

    [0-9]

    와 같습니다. 예를 들면,

    /\d/

    는 'No.1'의 '1'에 대응됩니다.

    \D

    		 숫자 문자가 아닌 문자에 대응되며,

    [^0-9]

    와 같습니다. 예를 들면,

    /\D/

    는 '2Pianos'의 'P'에 대응됩니다.

    \f

    		 폼피드 (U+000C) 문자에 대응됩니다. 폼피드 문자는 프린터에서, 연속용지를 다음 페이지의 시작 부분으로 넘기는 문자입니다.

    \n

    		 줄 바꿈 (U+000A) 문자에 대응됩니다.

    \r

    		 캐리지 리턴 (U+000D) 문자에 대응됩니다.

    \s

    		 스페이스, 탭, 폼피드, 줄 바꿈 문자 등을 포함한 하나의 공백 문자에 대응됩니다.

    [ \f\n\r\t\v\u00a0\u1680\u2000-\u200a\u2028\u2029\u202f\u205f\u3000\ufeff]

    와 같습니다.

    \S

    		 공백 문자가 아닌 하나의 문자에 대응됩니다.

    [^ \f\n\r\t\v\u00a0\u1680\u2000-\u200a\u2028\u2029\u202f\u205f\u3000\ufeff]

    와 같습니다.

    \t

    		 탭 (U+0009) 문자에 대응됩니다.

    \v

    		 수직 탭 (U+000B) 문자에 대응됩니다.

    \w

    		 밑줄 문자(_)를 포함한 영숫자 문자에 대응됩니다.

    [A-Za-z0-9_]

    와 같습니다. (여기에 대응되는 문자를 단어 문자라고 합니다.)

    \W

    		 단어 문자가 아닌 문자에 대응됩니다.

    [^A-Za-z0-9_]

    와 같습니다.

    \n

    		 정규식 안 n번 괄호의 최근 일치 부분의 역참조 입니다. (n은 양의 정수여야 합니다.)
    예를 들면,

    /apple(,)\sorange\1/

    는 '"apple, orange, cherry, peach.'의 'apple, orange,'에 대응됩니다.

    \0

    		 Null (U+0000) 문자에 대응됩니다. 이 때, 다른 숫자를 뒤에 쓰면 8진 이스케이프 시퀸스를 의미하게 됩니다.

    \xhh

    		 코드가 hh(2개의 16진 숫자)인 문자에 대응됩니다.

    \uhhhh

    		 코드가 hhhh(4개의 16진 숫자)인 문자에 대응됩니다.

    위에도 언급되었듯이, 정규식은 객체이기 때문에 메서드와 속성이 존재합니다:

    이름 설명

    lastIndex

    		 정규식의 검색을 시작할 인덱스를 반환하거나 지정합니다.

    flags

    		 정규식의 플래그를 반환합니다.

    global

    		 정규식에

    g

    플래그가 있으면

    true

    , 아니면

    false

    를 반환합니다.

    ignoreCase

    		 정규식에

    i

    플래그가 있으면

    true

    , 아니면

    false

    를 반환합니다.

    multiline

    		 정규식에

    m

    플래그가 있으면

    true

    , 아니면

    false

    를 반환합니다.

    sticky

    		 정규식에

    y

    플래그가 있으면

    true

    , 아니면

    false

    를 반환합니다.

    unicode

    		 정규식에

    u

    플래그가 있으면

    true

    , 아니면

    false

    를 반환합니다.

    compile(pattern, flags)

    		 스크립트 실행 중 정규식을 (다시) 컴파일합니다.

    exec(String str)

    		 문자열

    str

    중 대응되는 문자열에 대한 정보를 가지는 배열을 반환합니다. 대응되는 문자열이 없을 시, null을 반환합니다.

    test(String str)

    		 문자열

    str

    에 대응되는 문자열이 있는지 검사합니다. 있을 시

    true

    , 없을 시

    false

    를 반환합니다.

    toString()

    		 정규식을 문자열로 반환합니다.

    또한, 정규식에는 '플래그'를 통해 더 정확한 검색을 할 수 있게 해줍니다:

    이름 설명

    g

    		 어떤 정규식에 대응하는 모든 문자열을 검색합니다.

    i

    		 대소문자를 구분하지 않고 검색합니다.

    m

    		 다중 행 검색을 합니다.

    u

    		 패턴을 유니코드 코드 포인트의 나열로 취급합니다.

    y

    lastIndex

    메서드로 지정된 인덱스에서 부터 검색합니다.

    내장 객체

    내장 객체는 JavaScript 내에 기본적으로 내장되어있는 객체입니다. 내장 객체(그리고 객체 생성자 함수, 라이노 엔진>함수 편)를 사용하는 방법은 이와 같습니다:

    new 생성자(); // 내장 객체 또는 객체 생성자 함수를 사용할 때
    var 변수명 = new 생성자(); // 내장 객체 또는 객체 생성자 함수를 특정 변수에 참조할 때

    new 생성자();

    에서

    생성자()

    와 같은 것을 생성자라고 합니다. 생성자는 객체에 따라 다양하게 제공됩니다.



    또한, 정적 속성 또는 정적 메서드를 사용할 때에는

    new

    연산자를 사용해 객체를 생성한 다음 사용하지 않고,

    객체.속성

    또는

    객체.메서드()

    과 같이 사용합니다.

    Function객체는 함수를 선언하는 객체입니다.(*JavaScript에서는 함수도 객체 취급을 받을 수 있습니다.) 그리고 함수에 대한 속성 및 메서드를 제공합니다. (라이노 엔진>함수 편과 함께 보시는 것을 추천드립니다.)



    Function객체의 생성자는 이와 같습니다:

    Function(arg1, arg2, ... argN, functionBody);
    매개변수 뜻

    argN

    		 함수의 매개변수(함수의 기능을 구현하는데 필요한 추가정보를 의미하는 변수)를 의미하는 문자열 입니다. (선택적)

    functionBody

    		 함수의 기능을 구현하는 코드를 담은 문자열 입니다.

    그리고 다음은 Function객체의 속성 및 메서드입니다:

    이름 설명

    length

    		 함수의 매개변수 개수를 반환합니다.

    name

    		 함수의 함수명을 반환합니다.

    constructor

    		 객체의 프로토타입을 만드는 함수를 지정합니다.

    apply(thisArg, argsArray)

    		 함수를 호출하고

    this

    를 제공된 값으로 설정합니다. 매개변수는 배열로 전달될 수 있습니다.

    thisArg

    은 함수 객체를 호출하는데 제공될

    this

    의 값이고,

    argsArray

    는 함수 객체가 호출되어야 하는 인수를 지정하는 유사 배열 객체입니다. (함수에 제공된 매개변수가 없을 경우

    null

    또는

    undefined

    )

    bind(thisArg)
    bind(thisArg, arg1, arg2, ... argN)

    		 호출될 때

    this

    를 제공된 값으로 설정하는 새로운 함수를 만듭니다.

    call(thisArg, arg1, arg2, ... argN)

    		 주어진

    this

    값 및 각각 전달된 매개변수와 함께 함수를 호출합니다.

    thisArg

    은 함수 객체 호출에 제공되는

    this

    의 값이고,

    argN

    은 객체를 위한 매개변수입니다.

    toSource()

    		 함수의 소스코드를 문자열로 반환합니다. (

    Object.prototype.toSource

    를 재정의합니다.)

    toString()

    		 함수의 소스 코드를 나타내는 문자열을 반환합니다.

    Boolean

    Boolean객체는 참과 거짓, 즉, 불린 값을 감싸고 있는 객체입니다. Boolean객체의 생성자는 이와 같습니다:

    Boolean(value);
    매개변수 뜻

    value

    		 Boolean객체의 초깃값입니다.

    원시

    Boolean

    값과 달리,

    undefined

    와

    null

    을 제외한 모든 객체는 조건문에서

    true

    로 계산됩니다. 예를 들면,

    if (new Boolean(false)) {
        replier.reply("It is true.");
    }

    이 소스의 결괏값은

    > It is true.

    와 같습니다. 다음은 Boolean객체의 속성 및 메서드입니다:

    이름 설명

    toString()

    		 객체의 값에 따라 문자열

    "true"

    또는

    "false"

    를 반환합니다. (

    Object.prototype.toString()

    메서드를 재정의합니다.)

    valueOf()

    		 Boolean객체의 원시 값을 반환합니다. (

    Object.prototype.valueOf()

    메서드를 재정의합니다.)

    Symbol

    Symbol객체는 원시 데이터 형식 중 하나인 심볼(Symbol)에 관한 기능을 제공합니다. Symbol객체의 생성자는 이와 같습니다:

    Symbol(description);
    Symbol객체의 생성자는 new키워드와 함께 쓰이지 않습니다.
    매개변수 뜻

    description

    		 심볼에 대한 설명입니다. (선택적)

    심볼 값은 각각 고유한 값을 가지고 있습니다. 하물며 매개변수

    description

    의 값이 같아도 '서로 다름'으로 판정됩니다. 예를 들면,

    if (Symbol("hello") == Symbol("hello")) {
        replier.reply("It is true.");
    } else {
        replier.reply("It is false.");
    }

    이 소스의 결괏값은

    > It is false.

    와 같습니다. 다음은 Symbol객체의 속성 및 메서드입니다:

    이름 뜻

    length

    		 값이 0인 길이 속성입니다.

    prototype

    		 Symbol객체의 인스턴스에 속성을 추가할 수 있습니다.

    for(key)

    key

    로 현재 존재하는 심볼을 검색하고 찾으면 그것을 반환합니다. 존재하지 않으면

    key

    로 전역 심볼 레지스트리에 새로운 심볼을 생성하고 그 심볼을 반환합니다.

    keyFor(sym)

    		 전역 심볼 레지스트리로부터

    sym

    에 대한 공유 심볼 키(shared symbol key)를 추출합니다.

    toString()

    		 해당 Symbol객체를 나타내는 문자열을 반환합니다.

    valueOf()

    		 Symbol객체의 원시 값을 반환합니다.

    Error

    Error객체는 프로그램 실행 중에 발생되는 오류에 사용되는 객체입니다. Error객체의 생성자는 이와 같습니다:

    Error(message, fileName, lineNumber);
    매개변수 뜻

    message

    		 오류에 대한 설명입니다. (선택적)

    fileName

    		 생성할 Error객체의

    fileName

    속성 값. 기본값은 Error객체의 생성자를 호출한 코드를 포함하고 있는 파일의 이름입니다. (선택적)

    lineNumber

    		 생성할 Error객체의

    lineNumber

    속성 값. 기본값은 Error객체의 생성자 호출을 포함한 줄 번호입니다. (선택적)

    다음은 Error객체의 속성 및 객체입니다:

    이름 뜻

    prototype

    		 Error객체의 인스턴스에 속성을 추가할 수 있습니다.

    columnNumber

    		 오류가 발생한 줄의 열 번호를 반환합니다.

    fileName

    next()

    values()
